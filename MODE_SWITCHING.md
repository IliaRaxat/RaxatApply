# Переключение между тестовым и production режимами

## Тестовый режим (30 вакансий)
```bash
# Скопируйте тестовый файл в .env
cp .env.test .env
```

## Production режим (настраиваемое количество вакансий)
Вы можете выбрать количество вакансий для парсинга: 50, 300, 700 или 1000.

```bash
# Скопируйте production файл в .env
cp .env.production .env
```

Затем отредактируйте файл .env и измените значение VACANCY_COUNT:
```bash
VACANCY_COUNT=50    # или 300, 700, 1000
```

Или используйте удобный скрипт `SET_VACANCY_COUNT.bat`:
- Дважды щелкните по файлу и выберите нужное количество

## Проверка текущего режима
Посмотрите содержимое файла .env:
```bash
cat .env
```

## Автоматическое переключение
Вы также можете установить переменную окружения при запуске:
```bash
# Для тестового режима
TEST_MODE=true npm run dev

# Для production режима с 500 вакансиями
TEST_MODE=false VACANCY_COUNT=50 npm run dev

# Для production режима с 1000 вакансиями
TEST_MODE=false VACANCY_COUNT=1000 npm run dev
```
